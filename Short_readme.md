## 1. Что это такое

Скрипт автоматически развёртывает учебные стенды виртуальных машин на гипервизоре **Proxmox VE**. Он создаёт:

- **Пулы** — логические группы ВМ для каждого участника/стенда
- **Виртуальные машины** — из готовых шаблонов с образами дисков
- **Сетевые интерфейсы** — bridge-интерфейсы для внутренних сетей стенда
- **Пользователей** — учётные записи с ограниченными правами
- **Снапшоты** — для быстрого отката ВМ в исходное состояние

## 2. Системные требования

| Параметр | Минимум |
|----------|---------|
| PVE версия | 7.2+ (рекомендуется 8.0+) |
| Свободная ОЗУ | 6 ГБ (для загрузки образов) |
| Права | root |
| Подключение | SSH (рекомендуется) или Web Shell |
| Интернет | Нужен для скачивания образов ВМ |

## 3. Установка и запуск

### Быстрый запуск (скачать и запустить)

```bash
branch=main file=PVE-ASDaC-BASH.sh
curl -sOL "https://raw.githubusercontent.com/NiKeNO1540/PVE-ASDaC-BASH-nvkor/$branch/$file"
chmod +x $file
./$file
```

### Запуск с конфигурационным файлом

```bash
./$file -c my_config.conf
```

### Запуск в тихом режиме (без интерактивного ввода)

```bash
./$file -c my_config.conf -si -n 1-5 -var 1
```

При запуске вы увидите баннер:

```
Proxmox VE Automatic stand deployment and configuration script by PavelAF, Modified by NiKeNO1540
Original GitHub link: https://github.com/PavelAF/PVE-ASDaC-BASH
Modified GitHub link: https://github.com/NiKeNO1540/PVE-ASDaC-BASH-nvkor
```

## 5. Развёртывание стендов

**Пункт 1** — развернуть стенды из готовой конфигурации.

### Порядок действий:

1. **Выберите вариант установки** — скрипт покажет список доступных конфигураций с описанием и списком ВМ

2. **Укажите номера стендов** — какие стенды создавать:
   - `1-5` — создаст стенды 1, 2, 3, 4, 5
   - `1,3,7-10` — создаст стенды 1, 3, 7, 8, 9, 10
   - Каждый стенд — это полная копия конфигурации с отдельными ВМ, сетями и пользователем

3. **Проверьте и измените параметры** (при необходимости):
   - Внешний bridge-интерфейс (WAN)
   - Хранилище для дисков и ISO
   - Имена пулов и пользователей
   - Включение/выключение снапшотов
   - Создание учётных записей

4. **Подтвердите установку** — скрипт начнёт:
   - Скачивание образов дисков
   - Создание bridge-интерфейсов
   - Создание пулов и ВМ
   - Создание пользователей и назначение прав
   - Создание снапшотов

5. **Получите учётные данные** — логины и пароли для участников будут выведены в выбранном формате

### Пример вывода учётных данных:

```
Student-A1 | X7K9M
Student-A2 | B3R5W
Student-A3 | P8N2L
```

## 6. Создание своего стенда

**Пункт 4** — интерактивный мастер создания собственной конфигурации стенда.

### Шаг 1: Параметры стенда

Скрипт запросит:

| Параметр | Описание | Пример |
|----------|----------|--------|
| Описание | Название стенда | `Учебный стенд Linux` |
| Шаблон имени пула | `{0}` заменится номером | `MyStand-{0}` |
| Описание пула | Что будет написано у пула | `Стенд #{0}` |
| Учётные записи | Создавать ли пользователей | Да/Нет |
| Логин | Шаблон логина | `Student{0}` |
| Роль доступа | Какие права у пользователя | `Competitor_DE` |

### Шаг 2: Топология сетей

Определите внутренние именованные сети стенда:

```
Имя внутренней сети: ISP-HQ
✓ Добавлена сеть: ISP-HQ

Имя внутренней сети: ISP-BR
✓ Добавлена сеть: ISP-BR

Имя внутренней сети: HQ-Net
✓ Добавлена сеть: HQ-Net

Имя внутренней сети: [Enter — завершить]
```

Эти сети будут доступны для подключения ВМ.

### Шаг 3: Виртуальные машины

Для каждой ВМ:

**а) Базовые параметры:**
```
Имя ВМ: ISP
Шаблон: Alt-JeOS
Порядок запуска: 1
```

**б) Сетевые интерфейсы:**
```
Подключить к WAN? [y] — да
  Firewall на WAN? [y] — да
  DHCP firewall опция? [y] — да

Внутренние сети стенда:
  1. ISP-HQ
  2. ISP-BR
  3. HQ-Net
Номера сетей для этой ВМ: 1,2
```

Результат — ВМ `ISP` подключена к WAN, ISP-HQ и ISP-BR.

**в) Расширенные настройки сетей (при необходимости):**
- VLAN tag
- vlan_aware
- access_role для интерфейса

**г) Дополнительные диски и ISO (опционально)**

### Предпросмотр

```
═══ Предпросмотр конфигурации стенда ═══

Описание: Учебный стенд Linux
Пул:      MyStand-{0}
Сети:     ISP-HQ ISP-BR HQ-Net
ВМ (3):
  ISP [Alt-JeOS] сети: { bridge=inet, firewall=1 }, 🖧🖧: ISP-HQ, 🖧🖧: ISP-BR
  HQ-RTR [Alt-JeOS] сети: 🖧🖧: ISP-HQ, 🖧🖧: HQ-Net
  HQ-SRV [Alt-Server] сети: 🖧🖧: HQ-Net
```

### Сохранение и запуск

После подтверждения скрипт предложит:
1. **Сохранить в файл** — для повторного использования через `-c`
2. **Запустить развёртывание** — сразу создать стенды

При сохранении в файл выгружаются:
- Все шаблоны ВМ (`config_templates`)
- Роли доступа (`config_access_roles`)
- Конфигурация стенда со всеми ВМ

## 7. Управление стендами

**Пункт 2** — управление уже развёрнутыми стендами.

### Доступные действия:

| № | Действие | Описание |
|---|----------|----------|
| 1 | Включение учётных записей | Разрешить вход участникам |
| 2 | Отключение учётных записей | Заблокировать вход |
| 3 | Установка паролей | Сгенерировать новые пароли |
| 4 | Включить/перезагрузить ВМ | Массовый запуск |
| 5 | Выключить ВМ | Массовое выключение |
| 6 | Откатить к снапшоту Start | Сбросить все ВМ в начало |
| 7 | Создать снапшоты | Сохранить текущее состояние |
| 8 | Откатить снапшоты | Вернуться к сохранённому |
| 9 | Удалить снапшоты | Освободить место |
| 10 | Удаление стендов | Полное удаление (ВМ, пулы, сети, пользователи) |

### Типичные сценарии использования:

**Перед началом экзамена:**
1. Развернуть стенды (пункт 1 главного меню)
2. Включить учётные записи (2 → 1)
3. Раздать логины/пароли участникам

**Сброс стендов между попытками:**
1. Управление стендами (пункт 2)
2. Откатить к снапшоту Start (6)

**После экзамена:**
1. Управление стендами (пункт 2)
2. Удаление стендов (10)

## 8. Утилиты

**Пункт 3** — дополнительные твики для Proxmox VE.

| Утилита | Описание |
|---------|----------|

## 9. Работа с конфигурационными файлами

### Загрузка конфигурации

```bash
./PVE-ASDaC-BASH.sh -c my_stand.conf
```

### Выгрузка текущей конфигурации в файл

```bash
./PVE-ASDaC-BASH.sh -sh output.conf
```

### Выгрузка конфигурации с подробностями

```bash
./PVE-ASDaC-BASH.sh -sh output.conf -v
```

### Загрузка + очистка предыдущих конфигураций

```bash
./PVE-ASDaC-BASH.sh -z -c new_config.conf
```

### Структура конфигурационного файла

```bash
# Шаблоны ВМ — описывают базовые параметры ОС
config_templates[Alt-JeOS]='
    templ_descr = Шаблон ОС Альт JeOS
    os_descr    = Альт JeOS p10
    ostype      = l26
    cpu         = kvm64
    cores       = 1
    memory      = 512
    boot_disk_0 = https://example.com/disk.qcow2
'

# Вариант установки — стенд из нескольких ВМ
config_stand_1_var[stand_config]='
    description     = Мой стенд
    pool_name       = MyStand-{0}
    access_user_name = User{0}
'

config_stand_1_var[vm_1]='
    name            = Router
    config_template = Alt-JeOS
    startup         = order=1,up=8,down=30
    network_6       = { bridge=inet }
    network_7       = 🖧🖧: LAN
'

config_stand_1_var[vm_2]='
    name            = Server
    config_template = Alt-JeOS
    startup         = order=2,up=8,down=30
    network_6       = 🖧🖧: LAN
'
```

## 10. Аргументы командной строки

### Основные

| Аргумент | Описание |
|----------|----------|
| `-h`, `--help` | Справка |
| `-c FILE` | Загрузить конфигурацию из файла |
| `-sh FILE` | Выгрузить конфигурацию в файл |
| `-si` | Тихая установка (без вопросов) |
| `-n 1-5` | Номера стендов |
| `-var 1` | Номер варианта установки |
| `-v` | Подробный вывод |
| `--dry-run` | Тестовый запуск без реальных изменений |

### Параметры стенда

| Аргумент | Описание |
|----------|----------|
| `-st NAME` | Хранилище для дисков |
| `-iso NAME` | Хранилище для ISO |
| `-vmbr NAME` | Внешний bridge-интерфейс |
| `-vmid NUM` | Начальный VMID |
| `-pn NAME` | Шаблон имени пула |
| `-snap true/false` | Создавать снапшоты |
| `-acl true/false` | Создавать пользователей |
| `-u NAME` | Шаблон имени пользователя |
| `-l NUM` | Длина паролей |

### Примеры

**Развернуть 5 стендов варианта 1:**
```bash
./PVE-ASDaC-BASH.sh -si -c config.conf -var 1 -n 1-5
```

**Развернуть стенды на конкретном хранилище:**
```bash
./PVE-ASDaC-BASH.sh -si -c config.conf -var 1 -n 1-3 -st local-lvm
```

**Тестовый прогон без изменений:**
```bash
./PVE-ASDaC-BASH.sh -c config.conf -var 1 -n 1 --dry-run -v
```

## 11. Частые ошибки и решения

### «Не найдена команда qm/pvesh»

Скрипт запускается не на сервере Proxmox VE. Запустите на ноде PVE.

### «Недостаточно ОЗУ»

Образы дисков скачиваются во временный раздел tmpfs в ОЗУ. Нужно минимум 6 ГБ свободной ОЗУ. Выключите ненужные ВМ.

### «Пул уже существует»

Стенды с такими номерами уже развёрнуты. Либо удалите их (пункт 2 → 10), либо используйте другие номера.

### «Не удалось скачать файл»

Проверьте:
- Подключение к Интернету с сервера PVE
- DNS-настройки (`ping ya.ru`)
- Доступность ссылок на образы

### «Версия PVE устарела»

Минимальная версия — PVE 7.2. Некоторые функции (SDN, ACL для сетей) доступны только в PVE 8.0+.

### ВМ потеряли сеть после перезагрузки

Используйте утилиту «Фикс сетевых интерфейсов» (пункт 3 → 4). Особенно актуально для Alt Виртуализации.

### Как откатить все стенды к началу

```
Главное меню → 2 (Управление) → выбрать группу → 6 (Откатить к Start)
```

Выберите все стенды (Enter) — все ВМ будут сброшены к исходному снапшоту.
